技术文档写作规范（非原创）

发表于 2020-04-12 | | 阅读次数

技术文章与一般的写作不同，不需要迎合读者，将问题讲清楚自然能够吸引读者。

为了使文章更利于读者阅读，我们需要一套合理的写作规范，去标准化文章的逻辑结构与呈现效果。

一、写作原则

文章起始需增加，为什么我们需要 xxx ，为什么我选择 xxx 语言。

文章结束需增加，你学到了什么。

这些话语有助于读者系统地整理刚刚阅读获得的东西，也有助于作者本身对自己进行反思。

1. 多写，写的越多，会越容易，也会写的越好；

2. 言之有物。无需长篇累牍、文字臃肿，无需牵强、浮夸；

3. 做减法。将自己想表达的东西用最直接的方式表达出来，删除那些不必要的东西；

4. 反复阅读、朗读，反复修改，每次修改都是提高；

5. 拒绝新词，尤其是网络用语，有时候看起来幽默，其实是轻浮；

6. 多写短句子，偶尔长句，形成节奏感。当觉得需要节奏感，并开始注意到音节韵律的时候，写作能力可能就入门了；

7. 做加法。当你入门之后，就可以开始丰富文章内容了，并运用更多的写作技巧。

二、标题

标题往往是作者最先看到的东西，简明扼要的通过标题传递文章的内容信息，让读者第一时间知道内容对自己是否有用，感不感兴趣。技术文章的套路非常简单。

解决问题类的：XXX的解决方法、XXX的排查过程等

怎么做：如何XXX、怎么XXX等

介绍东西类的：什么是XXX、XXX简明介绍、区块链到底是什么等等

比如我要介绍VUE，就可以这样介绍标题《前端框架之VUE》

不要试图使用太花哨的标题，简单、快速传达出文章的重要内容，如果取了很花哨的标题，读者被内容诱导进来，内容干货多还好，过于平实容易消耗读者对你的信任。

五、内容

思路清晰、结构清晰、重点清晰。

1、 注意

主体内容之前先加一小段的概要，简明补充一下标题概括的内容

1. 重点内容重复；

2. 将有用信息提前；

3. 将简单的方案放在复杂少见的方案之前；

4. 操作类的，写明命令或者怎么操作，什么操作环境，明确指出预期结果

5. 合理加入图片，一图胜千言。

6. 尽量使用短句

7. 尽量使用简单句和并列句，避免使用复合句。

8. 用准“的”（形容词＋的＋名词）、“地”（副词+地+动词）、“得”（动词+得+副词）

2、三类主体内容

1）解决问题的文章

1. 问题是怎么发现的；

2. 排查问题的详细过程；

3. 总结反思；

4. 是否可以快速定位问题；

5. 如何再次避免问题。

2）怎么做的文章

1. 为什么要这么做；

2. 怎么做；

3. 做的过程会遇到哪些问题；

4. 分步说明；

5. 总结。

3）介绍类文章

1. 是什么？

2. 为什么是它？

3. 它的优点？

4. 它的缺点？

5. 它的原理？

6. 同类对比总结。

经常翻看，标点符号、错别字、代码错误、语病、文章样式这些都要注意，不要害怕观点错误或者技术错误，大胆的写，要写清楚。如果有人提意见，查资料核实，再修改，会提高很快。
