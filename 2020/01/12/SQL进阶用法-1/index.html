1. CASE表达式

1.1 CASE使用注意事项

  1. 在发现有WHEN的子句为真时，CASE表达式的真假值判断就会==终止==，剩余的WHEN子句会被忽略。因此，使用WHEN子句时需要注意条件的排他性。
  2. 统一各分支返回的数据类型。所有分支的返回数据类型必须一致，否则会报错。（经测试，MySQL中不报错）
  3. 不要忘记写END
  4. 养成写ELSE子句的习惯，方便明确判断逻辑

3 三值逻辑和NULL

  • SQL中的NULL与其他编程语言中的NULL是完全不同的东西，NULL既不是值也不是变量，知识一个表示“没有值”的标记,因此不能对NULL使用任何比较运算符得到的结果均为unknown。
  • SQL中逻辑值有三种结果：true、false、unknown
  • 逻辑值的AND和OR的值，优先级顺序为：
    1. AND的情况：false > unknown > true
    2. OR的情况：true > unknown > false
  • 如果NOT IN子查询中用到的表中被选择的列中存在NULL，则SQL语句的查询结果永远为空

    IN可以改写为多个=判断，然后用OR连接

  • EXISTS谓词永远不会返回unknown
  • IN 和 EXISTS可以相互替换，但是NOT IN 和NOT EXISTS有时不可以相互替换。这和AND和OR的优先级顺序相关
  • 当输入的表为空时，除count()外的聚合函数均会返回NULL

    SELECT AVG(age) FROM Student WHERE city = ‘北京’;
    当没有学生来自北京时，返回NULL

4 HAVING子句

  • SQL是面向集合的语言
  • 可以将表整体看做一个集合，不适用GROUP BY的情况下使用HAVING，HAVING子句可以单独使用，此时SELECT中不能列出表中的列，只能使用聚合函数和常量。
  • COUNT(*)查询的是所有行的数目，即使该行的所有值均为NULL
