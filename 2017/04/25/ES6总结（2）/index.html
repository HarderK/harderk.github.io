ES6总结（2）

发表于 2017-04-25 | 分类于 有关技术 | | 阅读次数

第二部分将总结下ES6中出现的新的API，同时我选择将新的原始数据结构的介绍也放在了这一部分，重点无疑在Promise和Generator部分，至于Buffer，我接触到的不多，只是在Node.js中有接触到，而且Node.js中的Buffer和ES6中的Buffer还有一些差异，在这个部分我就不进行介绍了。最后要提的一点是，新的类型和API在客户端使用前一定要先查浏览器支持的情况，可以去can i use查。

1. 新的原始数据类型

（1）Symbol

表示独一无二的值，通过Symbol()函数生成，不使用new进行实例化，因为其不是对象，let s = Symbol();
为了显示时比较方便区分，Symbol()可以接收一个字符串作为参数，let s1 = Symbol('test');打印出来是Symbol(test),但是即使描述字符串相同其代表的值也不同，描述字符串仅仅只是为了显示时便于区分。
利用Symbol.for(‘test’),可以使用同一个Symbol值,有该Symbol的时候则返回该Symbol，没有则创建。

(2) Set

类似于数组，但是成员值是唯一的，但是既不是数组也不是对象，是一种新的数据结构。

  • Set实例的属性和方法
  1. Set.prototype.constructor,不多说
  2. Set.prototype.size,成员总数
  3. add(value),添加值，返回Set结构，所以可以链式调用
  4. delete(value), 删除某个值，返回Boolean值
  5. has(value),检查是否含有某个值，返回Boolean
  6. clear(), 清除所有成员，没有返回值
  • 遍历
  1. keys()，返回遍历器对象
  2. values()，返回遍历器对象
  3. entries(),返回一个键值对的遍历器

以上三个方法在总结（1）数组扩展部分都有讲

  1. forEach(),是不是想起了什么，对就是ES5中数组的forEach()遍历方法。

由于Set只有值没有键，所以keys()和values()行为一致，可以理解为key和value是相同的

  • Set实例默认可遍历，默认遍历器生成函数就是它的values()方法for(let item of set){}

    (3) Map

    类似于对象,也是键值对的集合，但是各做类型（包括对象）都可以当做键

  • Map构造函数可以就接受一个数组作为参数，数组成员是表示键值对的数组

  • Map通过set(key, value)来添加成员，通过get(key)来获得成员的值

  • Map实例属性和操作方法

  1. size。成员的总数
  2. set(key,value)
  3. get(key)
  4. has(key)。返回Boolean
  5. delete(key)。删除某个键，返回Boolean
  6. clear()。清除所有成员
  • 遍历方法
  1. keys()
  2. values()
  3. entries()
  4. forEach()

Map实例的默认遍历器方法就是entries()方法for(let [key, value] of map){}

2. Iterator

Iterator(遍历器)是一种接口，为各种不同的数据结构提供统一的访问机制。

作用：

  1. 为各种数据提供一个统一的、简便的访问接口。
  2. 使得数据结构的成员能够按照某种次序排列。
  3. 供for...of遍历使用。

遍历过程：创建一个指针对象，指向当前数据结构的起始位置，第一次调用next方法，可以将指针指向第一个成员，接下来每次调用指向下一个成员，直到指向数据结构的结束位置。

  • ES6规定，默认的Iterator接口部署在数据结构的Symbol.iterator属性上，或者说一个数据结构只要有Symbol.iterator方法，就可以认为是可遍历的。调用Symbol.itetator方法就会得到得到遍历器生成函数。
  • 当使用for...of循环遍历某种数据结构时，该循环会主动寻找Iterator接口。

  • 在ES6中，有3类数据结构拥有原生的Iterator接口：数组、某些类似数组的对象（比如document.getElementsByClass()得到的类数组对象）、Set和Map结构.

  • 对象并没有部署Iterator接口，如果要使对象可以使用for...of遍历，就必须在Symbol.iterator属性上部署遍历器生成方法。

  • for…of循环遍历得到键值

3. 新的API

(1) Proxy

Proxy可以理解成在目标对象前架设了一个“拦截”层，外界对该对象的访问都必须通过这层拦截。就好像我们进行HTTP请求的时候，Client一般都不是直接与最终Server进行通信，而是中间要经过多个代理服务器。我们被墙也是因为中间的代理服务器在作怪。

(2) Reflect

Reflect对象有一下几个作用：

  1. 将Object对象上一些明显属于语言层面的方法都放到Reflect对象上，比如Object.keys() -> Reflect.kesy(),未来的新方法将只部署在对象上。
  2. 修改某些Object方法的返回结果，让其变得合理。比如Object.defineProperty(obj, name, desc)在无法定义属性时会抛出错误，而Reflect.defineProperty(obj, name, desc)则修改为返回false
  3. 让Object操作都变成函数行为。某些Object操作是命令式的，比如name in obj 和 delete name in obj; Reflect中定义了对应的Reflect.has(obj, name)和Reflect.deleteProperty(obj, name)
  4. Reflect对象的方法与Proxy对象的方法一一对应，不管Proxy怎么修改默认行为，你总可以在Reflect上获取默认行为。

一些Reflect对象的方法

  1. Reflect.setPrototypeOf(target, prototype); 类似于Object.create()
  2. Reflect.deleteProperty(target, name)
  3. Reflect.has(target, name);
  4. Reflect.set(target, name, value, receiver)
  5. Reflect.get(target, name, receiver)
  6. Reflect.apply(target, thisArg, args)
  7. Reflect.constructor(target, args)

(3) Generator函数

  • Generator函数可以理解为状态机，在其内部，有多个内部状态，执行到一个状态后就停止，等待下一步执行的指令。每一个yield语句就代表一个状态。

执行过程：首先调用Generator函数生成一个遍历器对象，然后不断调用遍历器对象的next()方法一步步的执行到下一个状态。

return语句只能执行一次，return之后的语句不再执行，return语句返回的done值为true。如果没有return语句，最后一条yield语句的done为false

  • next方法的参数，yield语句本身没有返回值，next()可以带一个参数，该参数会成为上一条yield语句的返回值。

  • for…of遍历Generator函数
    Generator函数生成的遍历器对象也有Symbol.iterator方法，执行后会返回自身。
    如果done的值为true，for…of循环就停止，且不包括done为true的返回值。

  • yield语句，用来在一个Generator函数中执行另外一个Generator函数

(4) Promise对象

Promise用来传递异步操作的消息。它代表了某个未来才会知道的结果
Promise有三个状态，Pending(进行中)、Resolved(已完成)、Rejected(已失败)
Promise状态的改变只有两种可能，从Pending到Resolved，从Pending到Rejectd。
Promise可以将异步操作已同步操作的流程表示出来，避免了层层嵌套的回调函数。

  • Promise构造函数,用来生成Promise实例

resolve()方法将Promise有Pending状态变为Resolved状态，reject()方法将Promise由Pending状态变为Rejected状态。
Promise实例生成后可以使用then指定Resolved状态和Rejected状态的回调函数

实例：

  • then(), then()方法返回的是一个新的Promise实例，因此可以使用链式调用

    then(function(){return something})方法中会将return的值转换成一个Promise实例，如何转换可以参考Promise.resolve()方法。
    如果return 的是Promise实例，则原样返回；如果转换的是基本值和非thenable对象，则返回新的Promise对象，且状态为Resolved,回调函数会立即执行；如果转换的是thenable对象，则返回一个新的Promise对象，该Promise的then()方法的行为与该thenable对象中then属性的行为一致。

  • Promise.prototype.catch()
    用于指定发生错误时的回调函数，一般不再then()的第二个参数制定发生错误时的回调函数，用catch()代替

  • Promise.all()
    用于将多个Promise实例包装成一个新的Promise实例，参数必须具有Iterator接口，一般是数组，数组成员是Promise实例，如果不是Promise，则通过Promise.resolve转换为Promise对象。
    几个要点：1. 只有每个Promise实例都变成Resolved，其状态才变成Resolved，且返回值为每个实例返回值组成的数组; 2. 只要有一个实例变成Rejected，则其变成Rejected，返回值第一个被reject()的实例的返回值。

  • Promise.race()
    与Promise.all()类似，只不过返回值不同，不同之处在于：
    只要有一个实例先改变，则Promise.race()的状态就跟着改变，看race这个单词就大概知道意思，就是谁快我就和谁的结果一样。

（2）继承

类之间通过extends关键字实现继承

super关键字指代父类的实例，自雷没有自己的this对象，而是继承了父类的this对象，然后对其进行加工，如果不调用super方法，子类就得不到this对象。
ES6的继承机制：先创造父类的实例对象this，然后再用子类的构造函数修改this，如果子类没有constructor，会默认添加。
